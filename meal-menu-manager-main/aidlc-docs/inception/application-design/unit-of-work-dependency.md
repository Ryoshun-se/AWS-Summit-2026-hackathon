## 推奨開発順序

### フェーズ 1: 基盤構築（並行不可）
| 順序 | ユニット | 理由 |
|---|---|---|
| 1 | Unit-0 共有インフラ | 全ユニットの前提条件 |
| 1 | Unit-6 iOS（フェーズ 1: モック開発） | Unit-0 と並行して先行開発可能 |

### フェーズ 2: バックエンド並行開発（Unit-0 完了後）
| 順序 | ユニット | 並行可否 | 備考 |
|---|---|---|---|
| 2 | Unit-1 認証・プロファイル | ✅ 並行可 | Unit-2 の前提だが独立して開発可能 |
| 2 | Unit-2 AI 献立生成エンジン | ✅ 並行可 | Unit-1 未完成時はプロファイルをモックで代替 |
| 2 | Unit-5 食事履歴・学習 | ✅ 並行可 | 他ユニットへの依存なし |

### フェーズ 3: 依存ユニット開発
| 順序 | ユニット | 前提ユニット |
|---|---|---|
| 3 | Unit-3 献立管理・レシピ | Unit-2 完了後 |

### フェーズ 4: 最終ユニット
| 順序 | ユニット | 前提ユニット |
|---|---|---|
| 4 | Unit-4 買い物リスト | Unit-3 完了後 |

### フェーズ 5: フロントエンド接続
| 順序 | ユニット | 内容 |
|---|---|---|
| 5 | Unit-6 iOS（フェーズ 2: API 接続） | 全バックエンドユニット完了後に実際の API に切り替え |

## クリティカルパス

```
Unit-0 → Unit-2 → Unit-3 → Unit-4 → Unit-6（接続）
```

最長の依存チェーンは Unit-0 → Unit-2 → Unit-3 → Unit-4 の 4 段階。Unit-1・Unit-5 はクリティカルパス外で並行開発可能。
