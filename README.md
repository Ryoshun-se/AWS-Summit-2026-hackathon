# 🍽️ 献立自動作成アプリ — 「人をダメにする」食事管理 AI

> **食事に関するすべての選択をアプリに委ねてください。あなたはただ、食べるだけでいい。**

---

## コンセプト

現代人は 1 日に最大 35,000 回の意思決定を行うと言われています。その多くは「今日の夕食は何にしよう」「冷蔵庫に何が残っているか」「子供は食べられるか」「先週も食べたっけ」といった、食事にまつわる無数の小さな選択です。これらの選択は一つひとつは些細でも、積み重なることで脳のリソースを消耗させ、本当に重要な判断の質を下げる「選択疲れ（Decision Fatigue）」を引き起こします。

食事における大変さの本質は **調理そのものではありません**。多くの人が疲弊しているのは、**「何を作るか」を決めること** と **「何を・どこで・いくらで買うか」を調べること** の 2 つです。そしてこの意思決定は、どれだけ上手くこなしても誰にも褒められることはありません。報われない意思決定を、毎日、何年も繰り返す。それが食事にまつわる現実です。

このアプリのコンセプトは **「人をダメにする」** です。

AI がすべての制約を満たした献立を自動生成し、食材を集計し、将来的には発注まで完結させる。完璧にこなされた意思決定が毎週届き続けると、人はやがて気づきます。**「自分でやる必要が、どこにもない」** と。大変で、複雑で、誰にも評価されない意思決定を一度アプリに委ねてしまえば、二度と自分でしようとは思わなくなるでしょう。献立を自分で考える能力は、使われないまま静かに錆びついていくかもしれません。それこそが、このアプリが目指す「ダメにする」の本質です。

## 主な機能

| 機能 | 説明 |
|---|---|
| 🤖 **AI 献立自動生成** | 毎週自動で 1 週間分の献立を生成。食材バランス・重複回避（外食・買い食い含む）・旬・家族の好みをすべて AI が考慮 |
| 👤 **ライフスタイルモード** | ファミリー / ボディメイク / ヘルスケア / ソロの 4 モードで、各ペルソナの制約条件を自動適用（PoC は単一モード選択。複数モード組み合わせは将来フェーズで検討） |
| 📅 **週間カレンダー表示** | 生成された献立を週間カレンダーで確認。気になる料理を AI に伝えるだけで代替案を自動生成 |
| ⚡ **即時リカバリー** | 外食・買い食い・献立変更を記録した時点で、残り週内の献立をモードごとの観点（PFC 帳尻・制約超過調整・余り食材優先・予算調整）で即時再生成。買い物リストも自動更新 |
| 🛒 **買い物リスト自動生成** | 確定した献立から食材を自動集計。重複食材は合算して表示。献立変更・即時リカバリー時も自動更新 |
| 📖 **レシピ詳細** | 各料理の材料・分量・調理手順を表示。人数に合わせて分量を自動調整 |
| 📝 **食事履歴・学習** | 実際に食べた料理を記録し AI が即時反映。献立通りなら記録のみ、変更・外食・買い食いなら週内献立を即時再生成。定番料理の長期学習にも対応 |
| 🚀 **将来: 自動発注** | ネットスーパー API 連携により、買い物リストからカート追加・注文まで自動化（将来フェーズ） |

## ターゲットユーザー

このアプリは「食事の選択行為に疲れているすべての人」を対象とします。

| ペルソナ | 主な課題 | アプリへの期待 |
|---|---|---|
| 👨‍👩‍👧 **子育て中の親** | 家族全員が食べられる献立を毎日考えるのが大変 | 子供対応・栄養バランスを自動で考慮した献立 |
| 💼 **共働き夫婦** | 仕事後に献立を考える余力がない | 何もしなくても週の献立が決まっている状態 |
| 💪 **ボディメイク実践者** | PFC バランスを保ちながらバリエーションを出すのが難しい | 筋トレ・リカバリーを考慮した高タンパク献立の自動生成 |
| 🏥 **生活習慣病のシニア** | 塩分・脂質制限の中で飽きない献立を考えるのが大変 | 医療的制約を守りながら多様な献立を自動生成 |
| 🎓 **一人暮らしの学生・社会人** | 食材が余る・栄養が偏る・食費が限られる | 1 人分・予算内・栄養バランスの取れた献立 |

## ビジネスモデル

**BtoC フリーミアム + 仲介料・広告収入**

| 収益源 | 内容 |
|---|---|
| 💳 **サブスクリプション（フリーミアム）** | 無料プランで基本機能を提供。プレミアムプランで高度な AI パーソナライズ・詳細な栄養管理・複数プロファイル等を提供 |
| 🏪 **連携スーパーからの仲介料** | 将来フェーズの食材発注機能において、連携先スーパーからの仲介手数料を収益化 |
| 📢 **広告収入** | 食品メーカー・スーパー等からのターゲティング広告（プレミアムプランでは非表示） |

## 技術スタック

| レイヤー | 技術 |
|---|---|
| **モバイル** | iOS（Swift / Flutter — 選定中） |
| **バックエンド** | AWS Lambda + API Gateway（機能別マイクロサービス） |
| **AI** | AWS Bedrock / OpenAI API（AI エージェント方式） |
| **データベース** | AWS DynamoDB |
| **認証** | AWS Cognito |
| **自動生成トリガー** | AWS EventBridge（週次スケジュール実行） |

## システムアーキテクチャ

```
[iOS アプリ]
    |
    | HTTPS / REST API
    v
[AWS API Gateway]
    |
    +-- AuthLambda          # 認証・アカウント管理
    +-- ProfileLambda       # プロファイル・ライフスタイルモード
    +-- MealGenerationLambda ← EventBridge（週次自動生成）
    |   MealGenerationLambda ← MealHistoryLambda（即時リカバリートリガー）
    |       |
    |       +-- MealPlanningAgent（AI エージェント）
    |               ツール: 履歴取得 / プロファイル取得 / 旬の食材 / 献立生成 / 週内残り再生成 / レシピ生成
    +-- MealPlanLambda      # 献立管理・確定
    +-- RecipeLambda        # レシピ詳細
    +-- ShoppingListLambda  # 買い物リスト（献立確定・即時リカバリー時に自動更新）
    +-- MealHistoryLambda   # 食事履歴・学習・即時リカバリートリガー
    |
[AWS DynamoDB]
```
